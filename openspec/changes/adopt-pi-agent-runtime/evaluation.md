## Evaluation Scope

- **Change:** replace the retained OpenNeko Agent/LLM/Skill/session path with Pi while preserving product-owned permission, task, resource, configuration, and credential boundaries.
- **Decision:** update existing indexed Agent runtime suites; do not create a general `pi-runtime` suite because the coverage index already assigns each affected behavior to a target owner.
- **Why real Evaluation is required:** the migration changes real prompt/Skill injection, Tool routing, effective provider/model configuration, queues, cancellation, tasks, session resume, and TUI event projection. Deterministic tests alone cannot prove that the complete TUI path uses Pi and rejects legacy fallback.
- **Canonical path:** TUI App/session owner → input queue → conversation-scoped Pi Agent → Pi provider or OpenNeko semantic Tool → Pi Session/product task authority → TUI projection.
- **Forbidden fallback:** legacy AgentSession/Executor/Think-Act loop, Platform chat/auth adapter, Vercel AI SDK chat glue, legacy Skill lifecycle, Journal transcript authority, nested model defaults, first-compatible model, implicit main-model fallback, mock provider, and direct turn runner.

## Evidence and Observability

Before suite updates, audit existing TUI facts and add only neutral runtime evidence that is missing from the owning contract:

- Agent runtime implementation identity and forbidden legacy-path participation;
- flat purpose key, effective provider/model identity, resolved-parameter digest, and immutable snapshot identity;
- Pi Session authority and matching conversation/session identity;
- program-owned user storage, virtual workspace locator, branch mapping, and turn checkpoint durability state;
- redacted Skill Host source/fingerprint and locator class, without physical path or raw Skill content in Evaluation facts;
- shared credential-store identity, credential source class, and auth mechanism, without token/key values;
- task/tool identities, terminal state, usage, cancellation, and dropped-fact counts.

Missing path evidence blocks the case. Final-answer text, model metadata alone, a dry run, or a mock must not substitute for runtime facts.

## Verification

- **Key-free validation:** run `pnpm test:agent:eval` after suite/schema/assertion changes, plus focused deterministic tests for Skill formatter path non-disclosure/transcript/cache behavior, flat policy normalization/fallback poisoning, Pi event/session/lease characterization, shared CredentialStore persistence/redaction, built-in OAuth, and Radius discovery when configured.
- **Selected dry runs:** validate every changed indexed case without provider-backed behavior before real execution.
- **Real cases:** run the same focused cases through the canonical TUI with available provider credentials and retain reports under the gitignored Agent Evaluation report root.
- **Blocked cases:** if credentials, model access, OAuth environment, network, or required runtime facts are unavailable, record `infrastructure-blocked` or the observability gap; do not replace the path with a mock or claim acceptance.

## Interpretation and Residual Risk

Success requires every canonical-path, effective-configuration, identity, task/session, and no-fallback hard gate to pass. Output quality is not the disputed contract, so no Judge is required. Key-free validation proves only harness and suite correctness; it is not real Pi behavior acceptance.

Residual risks to record at implementation time include Pi API/event/JSONL drift, Extension bundle/startup impact, unavailable real OAuth coverage, callback port and cancellation behavior, credential write failures, fact truncation, and accidental disclosure of absolute paths, cache handles, or secrets.
